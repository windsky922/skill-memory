# Conventions

## Communication

- Use Chinese when asking the user questions.
- Keep explanations concise and practical.
- State assumptions when making them.

## Files

- Preserve original thesis, report, data, and PPT files unless the user explicitly asks to modify them.
- Write generated deliverables under `outputs/` when no target path is specified.
- Use stable descriptive filenames.
- Keep temporary files under `tmp/`.

## Privacy

- Do not store secrets, API keys, private tokens, account credentials, or full raw sensitive datasets in memory files.
- Summarize reusable lessons instead of pasting large raw content.

## Verification

- For documents and slides, verify final outputs visually when possible.
- For data analysis, keep enough notebook/script context to reproduce results.
